# AGENTS.md

## 模板定位

- 本仓库是 `VibecodingTemplate`，用于在新项目开始时直接复制模板文件，快速进入需求澄清、设计、开发、测试和复盘闭环。
- 模板应保持通用、轻量、可复制；不要把某个具体业务项目的私有逻辑、账号、密钥、运行产物或临时数据写进模板。
- 默认文档语言使用中文；API 路径、协议字段、命令、代码标识、模型名、包名和原始报错保留英文原文。

## 作用范围

- 本文件约束模板仓库本身的维护。
- `template/` 目录中的文件是新项目启动时会被复制的默认内容。
- `template/AGENTS.md` 是新项目复制后的代理工作规则；调整新项目默认工作流时，优先更新 `template/AGENTS.md`。
- 根目录文档用于说明模板如何使用和维护，不应混入具体业务项目的实施记录。

## 维护原则

- 优先改进模板的默认结构、工作流、文档约定和可复制性。
- 新增目录或文件时，确认它对多数新项目都有复用价值；只服务单个项目的内容不要进入模板。
- 不预装大型框架、复杂脚手架或强绑定技术栈，除非用户明确要求模板面向该技术栈。
- 保持模板文件编码为 UTF-8，避免中文内容在 PowerShell、编辑器或复制后出现乱码。
- 不提交运行输出、日志、缓存、截图、构建产物、依赖安装目录或本机环境文件。

## 模板结构约定

- `template/AGENTS.md`：复制到新项目后的代理说明。
- `template/design/`：项目设计、阶段方案和阶段报告的默认位置。
- `template/research/`：调研记录、事实确认、依赖笔记和实验结论。
- `template/fix/`：问题修复记录，保留现象、根因、修复和验证闭环。
- `template/memory/`：项目内可复用经验和踩坑记录。
- `template/doc/`：面向使用者或开发者的说明文档。
- `template/requirements/`：运行或安装所需依赖文件、二进制、SDK 或扩展。
- `template/references/`：第三方 demo、协议样例、文档和工具包。
- `template/test/`：测试入口、测试脚本、测试素材和测试报告。

## 编辑规则

- 修改模板行为前，先判断目标是“维护模板仓库”还是“调整新项目默认规则”。
- 调整新项目默认规则时，同步检查 `template/AGENTS.md` 中的目录约定是否仍然一致。
- 不把生成项目后的业务决策写死在模板里；需要项目自行填写的内容使用清晰占位或说明。
- 遇到用户要求和模板约定冲突时，以用户当前明确要求为准，并在交付说明中点出差异。
- 对删除、迁移、批量重命名、依赖引入等影响较大的操作，先确认实际影响范围，再动手。

## 验证要求

- 修改 Markdown 后至少重新读取相关文件，确认内容完整、编码正常、没有占位错误或明显格式断裂。
- 修改目录结构后，使用 `rg --files` 或等效命令确认模板中没有意外产物。
- 若新增脚本或自动化入口，必须补充最小可运行说明，并执行一次可行的基础验证。
